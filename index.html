TINY_TOURS.EXE

STATUS: IN PROGRESS

Tiny Tours (UM RHELab)

> OVERVIEW

Tiny Tours is an ongoing VR research prototype focused on visualizing nuclear and radiation-related phenomena across multiple physical scales. The project explores how immersive VR environments can help users intuitively understand particle interactions, energy transfer, and radiation behavior that are traditionally represented through abstract equations and 2D diagrams.

> FEATURES

  • Multi-scale navigation between molecular, atomic, and subatomic environments relevant to radiation and nuclear processes
  • Interactive visualization of particle behavior and interaction events
  • Scale-aware VR interactions designed to preserve physical intuition across orders of magnitude
  • Overlays displaying particle type, relative energy, and interaction context
  • Free navigation using VR locomotion and hand-based interaction controls

> TECHNICAL NOTES

  • Built in Unity using C# and XR Interaction Toolkit.
  • Supports smooth movement between very different physical scales (molecular → subatomic)
  • Visualizes particle behavior and basic radiation-matter interactions in real time
  • Uses adjustable simulation speed to slow down or accelerate interactions
  • VR movement and hand interactions implemented with Unity XR Interaction Toolkit
BACKLINE_GAME.EXE

STATUS: FINISHED

Backline

> OVERVIEW

Backline is a third-person horror game I built in Unity, focused on atmosphere, exploration, and scripted interactions. The game is structured around moving through custom environments, interacting with objects, and triggering events that push the player forward. Most of the tension comes from how lighting, sound, and visibility are used to control what the player can see and anticipate.

> LORE

Andy is trying to find his friend Stark, who has gone missing. The game switches between two perspectives, with each one revealing different pieces of what happened.

> FEATURES

  • Third-person exploration with a focus on atmosphere and tension
  • Custom level design with intentional layouts to guide movement and visibility
  • Scripted interactions and trigger-based events (doors, props, audio, environment changes)
  • Audio cues that react to player movement, interactions, and progression
  • Lighting used to control mood and what the player can (or can’t) see
  • Scene transitions and checkpoints used to move the story forward
  • Two playable perspectives that switch between scenes as more of the story is revealed

> TECHNICAL NOTES

  • Built in Unity using C#, with reusable scripts for interactions, triggers, and events
  • Uses collider triggers and simple state checks to control progression and one-time scares
  • Layered audio setup with ambient loops, proximity sounds, and interaction SFX
  • Lighting tweaked per scene to balance mood and visibility
  • Set up so new scenes and events can be added easily using the same trigger and interaction system
HAND_TRACK.PY

STATUS: FINISHED

Hand-Tracking Driving

> OVERVIEW

This project started as an experiment to replace traditional keyboard controls with real-time hand tracking. Using only a webcam, the system tracks my hands and maps their movement to vehicle steering and braking. There are no controllers or sensors involved — just computer vision, vector math, and a lot of tuning to make it feel responsive and usable.

> FEATURES

  • Hands-only driving using real-time computer vision (no physical controllers)
  • Steering controlled by tilting both hands like an invisible steering wheel
  • Braking and reverse triggered through clenched hand gestures
  • On-screen visualization of the hand skeleton used by the tracking model
  • Adjusted vehicle physics so the car feels stable and controllable with gesture input

> TECHNICAL NOTES

  • Built in Unity using C# with Google MediaPipe for hand tracking
  • Tracks 21 hand landmarks per hand to estimate pose and gestures
  • Custom vector math used to convert wrist orientation into steering angles
  • Input smoothing applied to reduce jitter and improve control stability
  • Modified the Prometeo Car Controller to accept AI-based inputs instead of keyboard controls
TRAPPIST_SIM.EXE

STATUS: FINISHED

TRAPPIST-1 VR

> OVERVIEW

An interactive VR simulation of the TRAPPIST-1 exoplanetary system, designed to visualize tightly packed planetary orbits, compare relative planet sizes, and explore potential habitability.

> FEATURES

  • Explore all seven TRAPPIST-1 planets in VR with adjustable simulation speed.
  • View scientifically accurate relative planet sizes and orbital periods.
  • Side-by-side size comparison between Earth and each TRAPPIST-1 planet.
  • Per-planet information panels detailing planet type, orbital period, and habitability notes.
  • Free navigation using VR locomotion and interaction controls.

> TECHNICAL NOTES

  • Built in Unity using C#.
  • Implemented custom orbital motion using trigonometric functions (Mathf.Sin, Mathf.Cos) for smooth real-time planet movement.
  • Uses a global time multiplier to support fast-forward and slow-motion simulation.
  • VR interactions and locomotion managed via Unity XR Interaction Toolkit and XR Interaction Manager.
  • Planetary data sourced and cross-referenced from NASA's Eyes on Exoplanets TRAPPIST-1 simulation.
